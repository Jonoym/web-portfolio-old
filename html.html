Junstris

Introduction

This is a Java based tetris game with a fully functional interface and controls. The project was started during June of 2021 and was a good way of developing my skills with coding practices such as cohesion and coupling which is shown with the use of a model-view-controller system.


Challenges

Tetris controls vary depending on holding a control or tapping a control. Delayed auto shift or DAS is when holding a control down will cause repeated actions after a delay when first pressed. After experimenting with timers and system time, it was found that timers would cause unwanted affects such as double pressing so the system time method was chosen.


The game interface was difficult to design in a visually appealing way, previous iterations felt clunky or did not match which the overall appearance. The main page included very obvious tetris themes and includes useful animations on selection hover.


Since tetris is very timed focused, an accurate time needs to be displayed and updated regularly. This resulted in JavaFX canvases crashing regularly, although the issue was not fully elimnated, simplifying methods and only updating when necessary helped reduce the frequency.


Moving Forward

This is my first large coding project that I've carried out and provided me greater insight into integrating interfaces and models. I had to make decisions front start to finish and I got to experience the process firsthand and everything that goes into making it. Although it is only a small project, it increased my confidence in writing code and knowing that coding is capable of a lot of things.

Web Portfolio

Introduction

This web portfolio was created as a way to practice HTML, CSS and JavaScript in a way that can show my understanding. I first started following tutorials on specific parts of websites and gradually transitioned into turning the ideas into my own and joining them together.


Challenges

Since I was new to these languages, many errors that occured were due to not knowing proper syntax or making typos that were looked over. Overall, the project helped strengthen my understanding of the languages to a comfortable level.


Although I had plenty of portfolios to look over for inspiration, the web portfolio was for me and should also represent me. While I had many ideas, sometimes they were cut short by my lack of experience in the languages and not knowing if the idea had potential.


There are many ways to do something in any type of coding language and although everything works, it may not be optimal. Since I had nothing to reference to and reflect my code upon, over time, my code became very messy and somewhat unattainable.


Moving Forward

Going from small tutorials and websites to an animated one with multiple sections, I learnt the importance of good structuring quickly and how garbage code leads to propogating bugs. From this project, I was able to learn a language I was interested in and I have something that is somewhat presentable to show for it.
